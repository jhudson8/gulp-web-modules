So you've gotten this far... this is what you have.

You have the following directories now

public
where this index.html file is. All resources that are to be copied to the build root should be placed here.
config
deployment specific config data will be injected into your application available as global.config. When running the dev server, the config data used is config/dev.json
lib
any files placed here will have their content be injected into the application directly without any scoping
sections
the root of all section content within your application. Each directory under here will represent a unique section. Each section should have an index.js which will be the entry point (executed when the section is imported into the running app). All of the javascript files will be combined to a single javascript file (sections/{section name}.js). The section called base is a special section name that is required. sectiions/base/index.js will be your applicatin entry point (it will be executed when you include sections/base.js from your html container page.

Below is a message that is injected into this page using javascript. The point is to demonstrate the capabilities in a simple way. This is how it works:

  • sections/base/index.js uses the require to synchronously retrieve the exports of the sibling message-retriever module.
  • sections/base/message-retriever.js exports a function called getMessage which returns global.config.message
  • global.config.message comes from the injected configuration file config/dev.json
  • sections/base/index.js uses the requireSection to asynchronously retrieve the exports from another section (called foo). These exports contain a function called formatMessage. The return value is set in this html page.
  • sections/foo/index.html exports the formatMessage function as a section export using section.exports
